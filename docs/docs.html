Installation

Crew builds on macOS, Linux, and Windows.

One-liner (macOS & Linux)

curl -sSfL https://raw.githubusercontent.com/ashishtyagi10/crew/main/install.sh | sh

Installs the prebuilt crew binary to ~/.local/bin. Set INSTALL_DIR to choose another location.

With cargo (any platform with Rust)

cargo install --git https://github.com/ashishtyagi10/crew crew-app

GitHub Releases

Download the latest archive for your platform from the Releases page, extract it, and move the crew binary to a directory on your PATH:

Build from Source

git clone https://github.com/ashishtyagi10/crew.git cd crew cargo build --release -p crew-app # Binary is at target/release/crew

Updating

Type /update in the input bar to fetch and install the latest release — progress streams into an UPDATE card in the left nav — then /restart relaunches Crew into the new build whenever you're ready.

Build & Run

cargo run --release -p crew-app

Shells launch as your $SHELL login shell, so your full config and plugins load. The window opens with panes auto-tiled into a near-square grid and a docked command bar at the bottom.

crew starts detached by default — it re-launches itself in a new session and returns your prompt, so closing the launching terminal doesn't kill the window. Use crew --no-detach (or --foreground) to stay attached, e.g. to watch logs.

Panes

Panes auto-tile into a near-square grid. Each pane has a title bar (top row) showing its index, the program-set title (often the cwd), and right-aligned status glyphs. The focused pane has a near-white border and a bright block cursor; unfocused panes are grey with a dim cursor.

GlyphMeaning
⇡Nviewing scrollback, N lines back from the live bottom
●new output in an unfocused pane
!the program rang the bell
»receiving broadcast (synchronized) input

Click a pane to focus it (click the input bar to focus that); double-click a pane to toggle zoom. Inside a terminal pane, all other keys (arrows, Home/End, PageUp/Down, Ctrl+C, Shift+Tab, …) pass through to the program.

Crew shows up to 6 panes as full tiles; extra panes demote to a minimized thumbnail strip along the bottom. Any tile can also be minimized into the left nav with the [-] button on its top border — it keeps running but leaves the grid, and its sidebar PANES row gains a [+] to restore it (focusing a hidden pane always restores it).

Background panes can still flag you down: a bell (Claude Code prompting for input), a watched output pattern, or a long command finishing raises an attention marker on the pane's nav row and thumbnail — ! / ⚑ / ✓ in the bell colour, blinking for a few seconds and then holding until you focus the pane.

The Input Bar

The docked command bar at the bottom focuses with Cmd+I. The bar's legend shows Crew's current working directory (~-abbreviated).

Smart Routing

Bare text routes by context: if the focused pane is a visible idle shell, the line is typed into it; otherwise a first word that resolves on your login shell's $PATH spawns the command in its own pane, and anything else gets a hint instead of a mis-fire. The palette shows a one-row preview of exactly where Enter will send the line before you press it.

PrefixAction
!<cmd>Always run the command in its own new pane, regardless of focus
*<text>Broadcast one line to every terminal pane (one-shot alternative to Cmd+S broadcast mode)
?<plain english>Ask the AI for a command (à la Warp AI) — the suggestion lands back in the input bar, ready to edit or Enter
??<question>Ask the AI about the focused pane's output — the markdown explanation opens in the zoomed viewer

Autosuggest & History

  • Autosuggest — fish-style ghost text from history; Tab/→ accepts it.
  • History — Up/Down recall previous lines, persisted to $XDG_CONFIG/crew/history across sessions.

cd Completion

Typing cd <partial> ghost-completes the first matching subdirectory; Tab/→ accepts it. $VAR/${VAR} are expanded (e.g. cd $HOME/src). cd - toggles back to the previous directory, and the working directory is restored on the next launch. New shells (Cmd+T / /shell) open in the bar's current directory.

Editing & Status

  • Ctrl+W deletes the last word; Ctrl+U clears the line.
  • Status flashes — transient messages (e.g. "copied 12 lines", "cd: no such directory") appear briefly on the input card's bottom border.

Markdown

Crew renders markdown natively — a pulldown-cmark-based engine drawn straight to GPU cells: headings, lists, tables aligned by display width, fenced code as bordered cards, and clickable links.

  • Chat panes render agent replies as formatted markdown by default; Ctrl+Shift+M flips the focused chat pane to the raw source and back, and Cmd+Click opens a rendered link.
  • /md <file> opens a zoomed markdown viewer over one file, split into side-by-side source | preview halves with independent scroll: Tab switches the active half, arrows / PageUp / PageDown scroll it, r reloads from disk, Cmd+Click opens links in the preview, Esc closes.

Multi-Agent (/crew)

/crew opens a pane where independent headless CLI coding agents — claude, codex, and opencode — message each other to work a task you give them, in any direction.

On open, the broker probes which agent CLIs are installed and registers only those (missing ones are skipped, and it says so). Type a task and press Enter; prefix @<agent> to choose who starts, or @a+b to fan the task out to a subset in parallel. With no agent CLIs installed, the broker runs its inbuilt API agents (planner / coder / reviewer) over DashScope, OpenRouter, or Anthropic — whichever key it finds — and plugin-agent manifests make the pane work with no API key at all.

The relay protocol

Each agent receives a clean, normalized message — never another agent's raw CLI output — plus the task, its peers' capabilities, and the transcript so far. It ends its reply with a control line:

  • @next <agent> to hand off to a peer;
  • @done (or @done: <answer>) to end the thread — the explicit no-reply signal.

Parsing tolerates markdown wrappers, and a missing control line is re-asked once. This proves out as A→B (claude hands to codex), B→A (codex replies back), and a 3-way relay (claude → codex → opencode, with the answer relayed back to claude).

Constructs

Inside the pane, /-lines drive the broker itself (Tab completes @agents and /constructs; one-letter aliases and did-you-mean included): /fan sends one task to every agent in parallel, /loop <n> iterates on the crew's own answer, /goal works until a judge agent rules the goal met, /model pins agents to different models side by side, /plan / /approve / /reject is plan mode, /checkpoint / /checkpoints / /restore snapshot and restore the working tree under refs/crew/, /skill runs a prompt playbook, /mcp lists MCP tools, /export writes the transcript to Markdown, /compact folds old messages away, and /status reports live totals. Long constructs run as concurrent background tasks (default cap 4) tagged with #N chips — /tasks lists them, /stop [#n] cancels one or all.

Extending — skills, plugin agents, MCP, sys tools

  • Memory — Claude Code-style # shortcut: #always use pnpm appends to ./.crew/memory.md and every task thereafter carries the standing memory block (user + project, 2 KB cap); /memory shows what's loaded.
  • Skills — markdown playbooks in ~/.config/crew/skills/ or ./.crew/skills/ (flat .md or a directory with a SKILL.md); oversized playbooks disclose progressively as an outline agents read in chunks. /skill <name> <task> runs the relay with the playbook prepended.
  • Plugin agents — a JSON manifest in ~/.config/crew/agents/ or ./.crew/agents/ turns any headless CLI into a roster agent.
  • MCP servers — declared in mcp.json (standard mcpServers schema), connected lazily over stdio; agents call tools mid-relay with @tool server:tool {…}, bounded rounds, every call visible in the transcript.
  • Built-in sys tools — sys:run (bounded non-interactive shell), sys:read_file (chunked 64 KB reads), sys:write_file, sys:list_dir; CREW_SYS_MODE=readonly blocks the mutating pair.

All three drop-in surfaces hot-reload — edits are picked up live, and /reload forces it — no restart needed.

Guardrails

  • Loop guard — every message carries a hop counter; past the limit (default 6) the broker drops the thread and logs that it stopped, so a relay can never loop forever.
  • Per-call timeout — a hung agent is killed and logged (default 180s); the broker moves on instead of blocking.
  • Token budget — an optional per-thread cap (CREW_BROKER_TOKEN_BUDGET) terminates a thread that blows past it.
  • Isolation — agents run in a broker subprocess over Crew's JSON plugin protocol, so slow calls stay off the render thread; an adapter normalizes each agent's stdout before it is shown or relayed.
  • Observability — every hop is logged in the pane as from → to with the reply, under statusline-style agent rows with live context-window and time-share bars.

Adding an agent

Discovery and routing are agent-agnostic. To add a fourth agent, write one adapter constructor in crates/crew-plugin/src/broker/agents.rs and register it in known_adapters — the routing engine is untouched. Or skip the code entirely and drop a plugin-agent manifest.

Swarm Orchestration (crew-hive)

The /crew relay is a few CLI agents talking turn-by-turn. crew-hive is the next tier: a headless orchestration engine for running many agents toward a single goal — the substrate behind Crew's "command a fleet of agents" direction. It is a standalone workspace crate (no GPU, no terminal), driven by crew-app.

A goal is decomposed into a task-graph, executed over a bounded pool of agents, and the results merge upward while live telemetry streams out for the swarm view:

  • Planner — turns a goal into a dependency DAG (StubPlanner, or LlmPlanner that asks an LLM for the graph as JSON).
  • Scheduler — a tokio DAG executor with a bounded worker pool (concurrency cap), dependency fan-in/fan-out, failure cascade-cancel, panic-as-failure resilience, and cooperative cancellation.
  • Agents — a uniform Agent trait: StubAgent (tests), ApiAgent (a native LLM call — just a future, no PTY, so thousands can run), and RemoteAgent (dispatched over a wire to an out-of-process worker or an external engine such as LangGraph).
  • Blackboard — agents read their dependencies' results and write their own, merging work upward.
  • Bring-your-own-LLM — a Provider abstraction (mock + an Anthropic client) with per-agent ModelTier cost tiering (haiku / sonnet / opus).
  • Two modes, one engine — single-goal decomposition and flat parallel-job batches (batch_graph); a budget_governor enforces a hard cost ceiling.
  • Swarm view — a live task list over fleet telemetry: one row per task with a state glyph (○ pending · ● running · ✓ done · ✗ failed), its title, and the agent's last output line while it works.

The engine is wired into the app through two commands, each opening a live swarm pane (the task list + a live / done / failed / cost HUD, redrawn every frame): /goal <text> plans a goal off the UI thread then runs it — using the real LlmPlanner + ApiAgent workers when ANTHROPIC_API_KEY is set, else a deterministic offline stub backend; and /batch <file> runs a file of jobs (one per line) as a flat parallel swarm. Real-LLM runs are capped by the budget_governor (default $1.00). See ARCHITECTURE.md for the engine diagram.

Clipboard

  • Cmd+C copies the focused terminal's visible screen to the system clipboard.
  • Cmd+V pastes into the focused surface (terminal, input bar, or chat). For terminals it uses bracketed paste when the program enabled it.
  • Programs can copy to the system clipboard via OSC 52.

Scrollback

Mouse wheel or Shift+PageUp/Shift+PageDown scroll a pane's history; an amber ⇡ in the title bar marks that you're viewing scrollback. /find <text> scrolls back to the most recent line containing the text (smart case: case-insensitive unless the term has an uppercase letter); a miss reports on the status line. Typing returns to the bottom.

Settings

/settings opens a two-column bento form covering every configurable property (one column when the pane is narrow); Cmd+S / Alt+S saves:

  • Appearance — font family (type-to-search over installed monospace families, verified by measurement), font size, theme, accent (#rrggbb), paper texture + grain.
  • Window — nav width, show nav, launch maximized.
  • Notifications — the master switch, per-event toggles (cmd done, bell, pane exit), the min-secs threshold, and watched output patterns (one per line).

Settings persist to $XDG_CONFIG/crew/config.toml and apply live on Save; /restart picks up edits made outside the form.

Themes

Crew ships nine themes: five paper/ink looks — paper-dark (default, a high-contrast "newspaper" page), paper-light (a warm paper page), sepia-dark, midnight-ink, and graphite — and four CRT phosphor tubes: crt-green, crt-amber, crt-blue, and crt-violet, each a neon monochrome glow on a near-black tube.

Switch with /theme <name> (the palette offers an arrow-selectable picker) or cycle live with Ctrl+Shift+L; the choice persists. /theme random rotates through the dark themes every 10 minutes. Light themes render ink at Medium weight over 3× "newsprint" grain so they read like print; a faint GPU grain + vignette sits behind every theme (it reads as tube glow on the CRT ones), and every palette is picked for measured WCAG contrast.

Terminal panes answer OSC 10/11 color queries and export $COLORFGBG from the active theme, and a minimum-contrast floor (WCAG ratio 3.0, à la iTerm2) keeps program-painted text legible even after a live theme switch.

Architecture

Crew is a from-scratch GPU terminal. Every cell is drawn on the GPU; panes have SDF rounded borders. The terminal model is built on alacritty_terminal + portable-pty. In-pane UI (the settings form, command palette, and help overlay) is laid out with ratatui widgets into a Buffer and converted to GPU cells.

Crate Structure

Crew is a Cargo workspace with six crates:

CratePurpose
crew-appWindow, panes, input, in-pane UI
crew-renderGPU rendering (wgpu + glyphon)
crew-termPTY + terminal grid (alacritty_terminal + portable-pty)
crew-pluginChat / agent plugins (the /crew relay broker)
crew-themeTheme presets + palette contracts (9 themes, contrast thresholds)
crew-hiveSwarm orchestration engine (planner, scheduler, agent pool, blackboard, telemetry, remote/sidecar)

Hard Rules

  • Every .rs file stays ≤200 lines — no exceptions.
  • cargo clippy --workspace --all-targets is warning-free.
